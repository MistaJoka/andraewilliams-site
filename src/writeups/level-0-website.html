Problem

It's easy to jump straight into frameworks, component libraries, and deploy pipelines before understanding what a browser actually loads. A "Level 0" site fixes that: one folder you can open locally, trace every line, and change without tooling.

The portfolio itself uses a command-deck design system, JSON-driven sections, and platform CSS — but that polish sits on the same fundamentals this demo exposes.

What's included

  • index.html — home page with semantic structure and content sections.
  • about.html — second page to prove relative linking works.
  • style.css — layout, typography, color tokens, responsive rules, focus styles.
  • script.js — footer year stamp and active nav via aria-current.

Still excluded on purpose: build step, shared partials, backend, framework. Included in the demo: favicon, Open Graph, dark mode, print styles, and full a11y baseline.

Design choice

Level 0 uses a warm paper palette and system fonts — intentionally different from the operator deck. The contrast makes the boundary clear: this is the foundation; the main site is what you build on top.

Hosting

Source lives in src/level-0/ in the main repo. Vercel serves it at level0.andraewilliams.com via a host-based rewrite in vercel.json — same deploy as the portfolio, separate URL like promptbook.andraewilliams.com.

Requests to andraewilliams.com/level-0/ redirect permanently to the subdomain so the demo has one canonical home.

Verify locally

cd src/level-0
python3 -m http.server 8080
# open http://localhost:8080

Production: level0.andraewilliams.com after DNS is wired (see below).

DNS setup (manual)

  1. Vercel project → Settings → Domains → add level0.andraewilliams.com
  2. GoDaddy → CNAME level0 → cname.vercel-dns.com (or the target Vercel shows)
  3. Deploy main — rewrite rules ship with vercel.json
