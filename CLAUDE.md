# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

# Project Overview
This repo is a personal learning and portfolio project for Andrae.
The goal is to build real, modern web apps from beginner-to-intermediate level in a clean, production-style way.
Claude should prioritize teaching through practical implementation, not overexplaining theory.

# Current Goal
Build the first version of the app from a clean empty repo.
Start with the smallest working version first, then improve incrementally.

# Working Style
- Always propose a short plan before making major changes.
- Prefer small, reviewable changes over large rewrites.
- Explain tradeoffs briefly and clearly.
- When possible, teach through the code being written.
- Flag any assumptions before implementing them.

# Code Standards
- Keep files simple and readable.
- Prefer clear naming over clever naming.
- Do not introduce unnecessary abstractions early.
- Build the simplest version that works first.
- Add comments only where they help understanding.

# Workflow
- After making changes, explain what changed and why.
- Recommend the next best step after each completed task.
- Keep momentum high: do not stall on perfection.
- When the repo is missing structure, propose one before coding.
- Update docs when architectural decisions change.

# Project Structure Preference
- Keep docs in /docs
- Keep app code organized by feature or purpose
- Avoid clutter in the repo root
- Use a clean beginner-friendly structure

# Commands
- If commands are not yet available, help create them.
- Prefer commands that are easy to remember and standard for the stack.

# Definition of Done
A task is done when:
- the feature works
- the code is readable
- the files are placed correctly
- basic validation/testing is complete
- the next step is clear
